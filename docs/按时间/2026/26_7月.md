# 2026年7月

## 7月3号

今天遇到两个坑。

第一个问题是`toFixed`的取整问题，遵循四舍五入，这导致了数字格式化为指定小数位数的字符串后，多余的位数会进行四舍五入。

如果想保留小数又不想取整的话，可以使用`Math.trunc()`。

JS的数字转换需要特别注意下界限问题。

第二个问题是 Jenkins 和 阿里云 Codeup 。

我配置的公钥，只能拉取 A 项目，拉取不了 B 和 C 项目，排查用了很久的时间。因为我一直确信没有走部署公钥，而是个人公钥。

电脑本地存在多个密钥，而且私钥配置完成后一般查看不了。导致我也不确定 Codeup 用的是哪一个密钥。

最后解决的方法是通过在终端直接发起一次握手测试，强行让本地钥匙去跟云端对暗号。这次通过测试得知，本地访问 Codeup 的密钥和 Jenkins 上使用的密钥不是同一对。

正常的握手测试打印出来应该是`Welcome to Codeup, 你的名字!`，而部署公钥则是`Welcome to Codeup, Anonymous!`，由此得出结论。

查看本地所有的密钥信息

```bash
ls -la ~/.ssh/
```

握手测试

```bash
ssh -T git@codeup.aliyun.com

# 测试第一把钥匙（Ed25519 现代加密算法钥匙）
ssh -T -i ~/.ssh/id_ed25519 git@codeup.aliyun.com

# 测试第二把钥匙（如果你有老牌 RSA 钥匙的话）
ssh -T -i ~/.ssh/id_rsa git@codeup.aliyun.com
```
