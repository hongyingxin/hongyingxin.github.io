# 高阶组件

高阶组件（Higher-Order Component，简称 HOC）是 React 中用于复用组件逻辑的一种高级技巧。它不是 React API 的一部分，而是基于 React 的组合特性形成的一种设计模式。

## 定义

高阶组件是一个函数，它接收一个组件作为参数，并返回一个新的组件。

```js
const withLoading = (WrappedComponent) => {
  return (props) => {
    return props.isLoading ? <p>加载中...</p> : <WrappedComponent {...props} />;
  };
};
```

高阶组件与普通组件的区别：

- 普通组件： 将 Props 转换为 UI。

- HOC： 将一个组件转换为另一个组件。它不属于类组件或函数组件的特定类别，而是一种模式，既可以包装类组件，也可以包装函数组件

## 应用场景

横切关注点（Cross-Cutting Concerns）：将多个组件通用的逻辑抽离出来，避免代码冗余。

实际应用场景：

- 日志记录： 在组件挂载时上报埋点。

- 数据获取： 为组件自动注入来自 API 的数据。

- 样式统一： 为一系列组件包裹相同的布局或样式主题。

- 权限控制： 根据用户角色决定是否渲染组件。

## 实现

通常遵循 withNaming 的命名约定：

```js
function withLog(Component) {
  return class extends React.Component {
    componentDidMount() {
      console.log('组件已挂载');
    }
    render() {
      return <Component {...this.props} />;
    }
  };
}
```

**注意事项**

- 不要在 render 方法中使用 HOC： 这会导致组件每次渲染都重新创建，丢失状态并严重影响性能。

- 透传 Props： 必须确保将接收到的 Props 传递给被包装组件。

- 复制静态方法： 增强后的组件会丢失原组件的静态方法，需手动拷贝（或使用 hoist-non-react-statics 库）。

- Refs 不会透传： 需要使用 React.forwardRef 来解决。

## 属性代理和反向继承

### 属性代理

这是最常见的实现方式。HOC 接收 Props 并将其传递给被包装组件。你可以拦截、增加或修改 Props。

```js
// 例子：注入用户信息
function withUser(WrappedComponent) {
  return (props) => {
    const user = { name: "Gemini", role: "AI" };
    return <WrappedComponent {...props} user={user} />;
  };
}
```

### 反向继承

HOC 继承自 WrappedComponent。这种方式允许 HOC 通过 this 访问被包装组件的状态、生命周期和渲染方法。

**注意**： 这仅适用于类组件，且由于其高度耦合性，现在较少推荐。

```js
function withDebug(WrappedComponent) {
  return class extends WrappedComponent {
    render() {
      console.log('当前状态：', this.state);
      return super.render(); // 调用原组件的渲染逻辑
    }
  };
}
```

## 使用场景

### 状态管理

高阶组件可以作为“容器”，将复杂的状态逻辑提取出来。
```js
// 实现一个计数器逻辑的复用
function withCounter(WrappedComponent) {
  return function WithCounter(props) {
    const [count, setCount] = React.useState(0);
    const increment = () => setCount(c => c + 1);

    return (
      <WrappedComponent 
        {...props} 
        count={count} 
        increment={increment} 
      />
    );
  };
}
```

### 权限控制

权限控制是高阶组件最经典的使用案例。

```js
const withAuth = (requiredRole) => (WrappedComponent) => {
  return (props) => {
    const { userRole } = props; // 假设从全局或 Props 获取

    if (userRole !== requiredRole) {
      return <div>抱歉，你没有权限访问此页面。</div>;
    }

    return <WrappedComponent {...props} />;
  };
};

// 使用
const AdminPanel = () => <h1>管理员后台</h1>;
const ProtectedAdminPanel = withAuth('admin')(AdminPanel);
```
