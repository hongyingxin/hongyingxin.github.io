# 事件机制

React 的事件机制涉及 合成事件、事件代理、事件传播机制以及原生事件等多个方向。

## 事件系统

### 合成事件

React 并不直接使用原生DOM事件，而是采用合成事件进行封装：

- 统一事件处理，兼容不同浏览器
- 事件对象是 React 维护的对象，会被 事件池（Event Pooling）复用，提升性能
- 事件在所有浏览器中表现一致，避免浏览器差异带来的问题

```js
function handleClick(event) {
  console.log(event); // 这是一个 SyntheticEvent
  console.log(event.nativeEvent); // 获取原生事件对象
}

return <button onClick={handleClick}>Click me</button>;
```

### 事件代理

React 并不会直接把事件绑定到DOM元素上，而是采用事件代理机制：

- 事件会委托到最外层的 document，然后通过事件冒泡机制触发组件内的事件处理函数
- 这种方式可以减少事件绑定，提高性能，适用于大规模列表渲染的场景


```js
function App() {
  return (
    <div onClick={() => console.log("Parent Div Clicked")}>
      <button onClick={(e) => e.stopPropagation()}>Click me</button>
    </div>
  );
}
```

### 绑定方式

事件绑定可以是箭头函数、bind绑定、class方法

```js
class App extends React.Component {
  handleClick() {
    console.log(this); // 这里的 this 需要绑定
  }

  render() {
    return (
      <button onClick={this.handleClick.bind(this)}>Click Me</button>
    );
  }
}
```

### 事件传播

React的事件默认只支持冒泡，不支持捕获，但可以通过onClickCapture监听捕获阶段

```js
function App() {
  return (
    <div onClickCapture={() => console.log("捕获阶段：父元素")}>
      <button onClick={() => console.log("冒泡阶段：子元素")}>
        Click Me
      </button>
    </div>
  );
}
```

### 阻止

- stopPropagation()：阻止事件冒泡（但在React事件代理下，可能仍会触发document上的监听）
- preventDefault()：阻止事件默认行为，如submit提交表单

```js
function App() {
  return (
    <form onSubmit={(e) => e.preventDefault()}>
      <button type="submit">Submit</button>
    </form>
  );
}
```
