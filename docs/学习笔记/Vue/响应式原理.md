# 响应式原理

vue3源码采用了`monorepo`的方式进行管理，将响应式拆分成了一个单独的`reactivity`包。本次源码学习都是基于`reactivity`包进行的。

## 1. reactive

在Vue3中，reactive函数是进入响应式世界的入口。其核心逻辑集中在`packages/reactivity/src/reactive.ts`文件中。

reactive的本质是：将一个普通对象包装成Proxy对象，并为其配置一套“代理”。

### 1.1 reactive函数定义

`reactive`函数的主要职责是接收一个对象并返回其`Proxy`代理。

当我们使用`reactive`函数时，实际上是调用了`createReactiveObject`函数。

```js
export function reactive<T extends object>(target: T): Reactive<T>
export function reactive(target: object) {
  // 如果已经是只读代理，直接返回
  if (isReadonly(target)) {
    return target
  }
  return createReactiveObject(
    target,
    false,
    mutableHandlers,  // 普通对象的拦截处理器
    mutableCollectionHandlers,  // Map/Set等集合类型的拦截处理器
    reactiveMap,  // 响应式对象的缓存
  )
}
```

### 1.2 createReactiveObject核心函数

```js
function createReactiveObject(
  target: Target,
  isReadonly: boolean,
  baseHandlers: ProxyHandler<any>,
  collectionHandlers: ProxyHandler<any>,
  proxyMap: WeakMap<Target, any>,
) {
  // 1. 基础检查：只有对象才能被代理
  if (!isObject(target)) {
    if (__DEV__) {
      warn(
        `value cannot be made ${isReadonly ? 'readonly' : 'reactive'}: ${String(
          target,
        )}`,
      )
    }
    return target
  }
  // 2. 如果 target 已经是一个 Proxy（除非是 readonly(reactive) 情况），直接返回
  if (
    target[ReactiveFlags.RAW] &&
    !(isReadonly && target[ReactiveFlags.IS_REACTIVE])
  ) {
    return target
  }
  // 3. 检查缓存：同一个对象不要重复创建代理
  const existingProxy = proxyMap.get(target)
  if (existingProxy) {
    return existingProxy
  }
  // 4. 检查是否在不可代理白名单内（如被 markRaw 标记过的）
  const targetType = getTargetType(target)
  if (targetType === TargetType.INVALID) {
    return target
  }
  // 5. 【核心】创建 Proxy
  // 区分普通对象和集合（Map/Set），使用不同的拦截处理器
  const proxy = new Proxy(
    target,
    targetType === TargetType.COLLECTION ? collectionHandlers : baseHandlers,
  )
  // 6. 存入缓存
  proxyMap.set(target, proxy)
  return proxy
}
```

源码在这里做了多重校验，确保代理过程的高效和正确。

#### proxyMap(WeakMap)

proxyMap(WeakMap):这是全局的依赖关系管理，用于存储已经代理过的对象。

#### baseHandlers

在创建proxy实例时，对普通对象和集合（Map/Set）使用不同的拦截处理器。这里主要关注`baseHandlers`方法

### 1.3 baseHandlers

Proxy的强大在于它的第二个参数。上文提到，如果是普通对象，使用`baseHandlers`，这个参数是`mutableHandlers`。

源代码路径在`packages/reactivity/src/baseHandlers.ts`文件中。这里拦截了多种操作，最核心的是get和set方法。

**读取拦截**

```js
get(target: Target, key: string | symbol, receiver: object): any {
  // 1. 处理内部标识（如 __v_isReactive 等）
  if (key === ReactiveFlags.IS_REACTIVE) return !this._isReadonly
  if (key === ReactiveFlags.IS_READONLY) return this._isReadonly
  if (key === ReactiveFlags.RAW) return target

  const targetIsArray = isArray(target)
  // 2. 数组特殊处理（如 push, pop 等方法需要特殊拦截）
  if (!isReadonly && targetIsArray && (fn = arrayInstrumentations[key])) {
    return fn
  }

  // 3. 【核心】获取原始结果
  const res = Reflect.get(target, key, isRef(target) ? target : receiver)

  // 4. 【关键点】依赖收集 (Track)
  // 只有在非只读且 key 不是内置 Symbol 时，才进行追踪
  if (!isReadonly) {
    track(target, TrackOpTypes.GET, key)
  }

  // 5. 如果是 shallow（浅层响应式），直接返回结果
  if (isShallow) return res

  // 6. 【精髓】处理嵌套对象（懒代理）
  if (isRef(res)) {
    // Ref 自动解包（如果是数组索引访问除外）
    return targetIsArray && isIntegerKey(key) ? res : res.value
  }
  if (isObject(res)) {
    // 只有当你真正访问到嵌套对象时，Vue 才会去代理它
    // 这就是 Vue 3 性能优于 Vue 2 的原因：不是一次性递归所有属性
    return isReadonly ? readonly(res) : reactive(res)
  }

  return res
}
```

**修改拦截**

```js
set(
  target: Record<string | symbol, unknown>,
  key: string | symbol,
  value: unknown,
  receiver: object,
): boolean {
  let oldValue = target[key]

  // 1. 性能优化：尽量存储原始值 (toRaw)
  if (!this._isShallow) {
    oldValue = toRaw(oldValue)
    value = toRaw(value)
    // 如果旧值是 Ref 且新值不是 Ref，直接修改 .value
    if (!isArray(target) && isRef(oldValue) && !isRef(value)) {
      oldValue.value = value
      return true
    }
  }

  // 2. 检查是“新增属性”还是“修改属性”
  const hadKey = isArray(target) && isIntegerKey(key)
      ? Number(key) < target.length
      : hasOwn(target, key)

  // 3. 执行真实的赋值
  const result = Reflect.set(target, key, value, receiver)

  // 4. 【关键点】派发更新 (Trigger)
  // 只有当 target 确实是当前代理的对象时（排除原型链干扰）且值发生了变化，才触发
  if (target === toRaw(receiver)) {
    if (!hadKey) {
      trigger(target, TriggerOpTypes.ADD, key, value)
    } else if (hasChanged(value, oldValue)) {
      trigger(target, TriggerOpTypes.SET, key, value, oldValue)
    }
  }
  return result
}
```

#### Reflect

set和get方法里面使用`Reflect`配合`receiver`参数，能确保当前对象在继承关系时，this始终指向当前的Proxy对象，从而保证依赖收集不会丢失。`target[key]`无法确保this。

#### 惰性代理

只在 get 时（即你真正用到这个属性时），才去判断这个属性是不是对象。如果是，再调用 reactive(res) 进行代理。这大大节省了不必要的开销。

### 总结

- 多重缓存：利用proxyMap(WeakMap)确保代理对象的唯一性（缓存已经代理过的对象）

- 分而治之：针对普通对象和集合类使用了不同的拦截方案（baseHandlers和collectionHandlers）

- 惰性代理：将递归压力从初始化阶段分散到运行阶段（get时才进行代理）

## 2. Effect

在源码`packages/reactivity/src/effect.ts`文件中，effect被称为副作用函数。

watch、computed、在模板使用变量，Vue内部都会把这些代码包装成一个effect。

### 2.1 activeEffect

```js
export let activeSub: Subscriber | undefined
```

activeSub是整个响应式系统的灵魂，用来记录当前运行中的effect函数。

- 在全局，Vue维护了一个变量activeEffect
- 当effect.run()执行时，它会把自己赋值给activeEffect
- 这样，当代码运行到proxy.name触发get拦截时，track函数只需要看一眼全局的activeEffect，就能知道该把谁存进targetMap。

### 2.2 ReactiveEffect类

effect的底层是一个类。每一个effect(fn)都会创建一个ReactiveEffect实例。它的核心任务是执行fn，并在执行期间把自己“置顶”。

```js
run(): T {
  // 1. 检查是否处于活动状态
  if (!(this.flags & EffectFlags.ACTIVE)) {
    return this.fn()
  }

  // 2. 状态标记
  this.flags |= EffectFlags.RUNNING

  // 3. 【核心】将自己设为当前活跃的听众
  const prevEffect = activeSub
  activeSub = this

  // 开启依赖追踪开关
  shouldTrack = true

  try {
    // 4. 执行原始函数。
    // 重点：在执行 fn() 期间，它会访问响应式数据，从而触发 Proxy 的 get
    // 也就是上一关我们看到的 track 函数。
    return this.fn()
  } finally {
    // 5. 收尾工作：恢复之前的状态
    activeSub = prevEffect
    this.flags &= ~EffectFlags.RUNNING
  }
}
```

### 2.3 嵌套effect

在实际场景中，effect是会嵌套的（比如父组件嵌套子组件），如何确保activeSub的顺序呢？

Vue3内部使用类似于“栈解构”的逻辑。在 run() 方法中，Vue 通过 const prevEffect = activeSub 保存了上一个听众，在 finally 块中通过 activeSub = prevEffect 进行了恢复。

- 进入子effect前，把当前的父effect存起来
- 子effect执行完，把activeEffect恢复为父effect
- 这样，就确保了activeEffect的顺序是正确的。

### 2.4 调度器 scheduler

effect 还有一个重要的参数是 scheduler。

- 如果没有 scheduler，数据一变，effect 立即同步执行。

- 如果有 scheduler，数据变化时，Vue 会把执行权交给调度器。这正是 Vue 异步更新队列 的基础：它不会让你改 100 次数据就刷 100 次视图，而是缓存起来，等微任务阶段刷一次。

### 2.5 运行流程

- effect 运行 -> 设置 activeSub。

- 读取数据 -> 触发 Proxy.get -> 调用 track。

- track 拿到 activeSub -> 存入dep。

### 2.6 总结

- effect 是订阅者：它是最终被记录在 targetMap 里的那个东西。

- activeEffect 是桥梁：它让 track 知道当前是谁在读取数据。

- 执行即收集：只有在 effect 运行期间读取数据，才会被追踪。

## 3. dep

### 3.1 targetMap

这是整个 Vue 3 唯一的全局依赖存储中心。

```js
export const targetMap: WeakMap<object, KeyToDepMap> = new WeakMap()
```

它的结构是三级嵌套：

- 第一层 (WeakMap): target（对象本身）-> depsMap。
- 第二层 (Map): key（属性名）-> dep。
- 第三层 (Dep): 存储所有订阅者的实例（在 3.5 之前是 `Set<Effect>`，3.5 之后是 Link 链表）。

### 3.2 track 依赖收集

```js
export function track(target: object, type: TrackOpTypes, key: unknown): void {
  // 1. 只有当前有活跃的听众 (activeSub) 且允许追踪时才执行
  if (shouldTrack && activeSub) {
    let depsMap = targetMap.get(target)
    if (!depsMap) {
      targetMap.set(target, (depsMap = new Map()))
    }

    let dep = depsMap.get(key)
    if (!dep) {
      // 2. 为该属性创建一个专有的 Dep 订阅中心
      depsMap.set(key, (dep = new Dep()))
      dep.map = depsMap
      dep.key = key
    }

    // 3. 【点睛之笔】让这个 Dep 记住当前的 activeSub
    // 这就是“依赖收集”：把当前运行的函数记录在对应属性的档案下
    dep.track()
  }
}
```

### 3.3 trigger 派发更新

```js
export function trigger(target: object, type: TriggerOpTypes, key?: unknown, ...) {
  // 1. 根据对象找到其对应的 depsMap
  const depsMap = targetMap.get(target)
  if (!depsMap) return

  // 2. 根据 key 找到对应的 Dep 订阅中心
  const dep = depsMap.get(key)

  // 3. 【核心】通知所有订阅者
  if (dep) {
    // 这里的 dep.trigger 内部会遍历所有记录在案的 effect
    // 告诉它们：“数据变了，你们该干活了！”
    dep.trigger()
  }
}
```

## 4. 总结

1. reactive.ts用Proxy拦截数据
2. baseHandlers.ts用get和set拦截数据
3. effect.ts暴露activeSub，dep.ts的track依赖收集将他放到targetMap中
4. dep.ts的trigger派发更新，从targetMap获取并重新执行activeSub
