# 目录结构

ElementPlus 使用了 Monorepo（单仓多包）架构。这种架构能将大型项目拆解为多个职责明确的包，既方便代码复用，又方便独立维护。

## 核心包目录

packages 目录是组件库的灵魂，所有的核心代码都在这里。

- components/：组件实现。每个组件（如 Button、Input、Table 等）都有自己的文件夹，包含`.vue`、`.ts`和单元测试。

- theme-chalk/：样式仓库。专门存放SCSS文件。ElementPlus将样式和逻辑彻底分离，方便用户按需引入或主题定制。

- hooks/：通用钩子。存放跨组件复用的Composition API逻辑（如 useNamespace、useSize、useDisabled 等）。

- utils/：工具函数。存放纯JS逻辑，不涉及UI（如DOM操作、类型判断、对象处理）。

- constants/：共享常量。如组件尺寸定义、事件名称、ARIA属性等。

- directives/：自定义指令。如v-loading、v-permission等。

- locale/：多语言/国际化。存放各个国家的语言包。

- element-plus/：主入口包。它的作用是将上述所有包打包在一起，最后发布到npm的element-plus就是由它导出的。

## 开发与文档工具

docs/，play/：开发与文档工具。docs/是文档目录，play/是开发环境目录。

为了让开发者能够方便地编写代码，项目提供了两个核心环境：

- play/ (Playground)：一个极简的 Vue 应用。开发者在 packages/ 里改了代码，直接在 play 里就能看到效果。它是内部开发调试的第一现场。

- docs/ (Documentation)：基于 VitePress 的文档网站。你看到的官方文档、示例代码都存放在这里。

## 工程化支持

internal/，scripts/：工程化支持。internal/是内部工具库，scripts/是脚本工具。

- internal/build/：构建脚本。定义了如何把 TS/Vue 代码转换成浏览器能识别的 JS、如何提取 CSS、如何生成类型定义文件。

- scripts/：辅助脚本。比如自动生成新组件模板、同步版本号、发布流程脚本等。

## SSR渲染

ssr-testing/：专门用于测试组件在服务器端渲染时的稳定性

## 类型声明文件

typings/：存放全局的TypeScript类型声明文件(.d.ts)。

## 总结

以上就是ElementPlus主要的目录结构，体现出一个核心思想：职责分离。

- 逻辑与样式分离：逻辑在 packages/components，样式在 packages/theme-chalk。这样用户如果只想换皮肤，只需要替换 CSS 包即可。

- 通用逻辑复用：通过 hooks 和 utils 包，避免了在 100 多个组件里重复写同样的代码。

- 独立包依赖：如果你只需要 Element Plus 的工具函数，你可以只安装 @element-plus/utils，而不需要下载整个组件库。
