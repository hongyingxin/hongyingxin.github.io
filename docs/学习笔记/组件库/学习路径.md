# Element Plus 组件库深度学习路径 (Roadmap)

这是一份专为希望深入掌握工业级 Vue 组件库封装技术的开发者设计的学习路径。

---

## 📅 第一阶段：基础设施与架构 (Infrastructure)

_目标：理解 Monorepo 组织方式及开发环境搭建_

- [ ] **Monorepo 架构**：理解 `pnpm-workspace` 及其包管理逻辑（packages/\*）。
- [ ] **开发环境启动**：成功运行 `play` (Playground) 和 `docs` (文档)。
- [ ] **构建系统**：探索 `internal/build`，了解组件是如何从源码被打包成 lib/es 格式的。
- [ ] **Icon 系统**：研究 `@element-plus/icons-vue` 的独立封装与按需加载逻辑。

---

## 🎨 第二阶段：设计规范与标准 (Standards)

_目标：掌握组件库的“形”，即统一定义与导出规范_

- [ ] **BEM 命名规范**：掌握 `useNamespace` (JS) 与 SCSS Mixins 的配合。
- [ ] **组件定义标准**：学习 `buildProps` 的强类型 Props 定义方式。
- [ ] **插件化导出**：理解 `withInstall` 逻辑，掌握全局注册与按需引入的原理。
- [ ] **TypeScript 增强**：学习如何生成全局组件类型声明（`global.d.ts`）。

---

## 🧠 第三阶段：逻辑抽离与 Hook 化 (Logic & Hooks)

_目标：掌握组件库的“神”，即复杂的交互与环境感知逻辑_

- [ ] **响应式尺寸感知 (Size System)**：研究 `useSize`、`useFormSize`。了解组件如何获取尺寸（Props -> Form -> Global Config）。
- [ ] **状态感知 (State System)**：研究 `useDisabled`、`useFormDisabled`。
- [ ] **唯一标示管理**：学习 `useId` 的生成逻辑（特别是在 SSR 场景下的处理）。
- [ ] **交互 Hook**：研究 `useClickOutside`、`useFocus`、`useZIndex` (弹窗层级管理)。

---

## 🌍 第四阶段：全局管理与上下文 (Context & Global)

_目标：掌握组件库的“脑”，即全局控制能力_

- [ ] **Config Provider**：理解 `provide` / `inject` 在组件库中的高级应用。
- [ ] **国际化 (i18n)**：研究 `useLocale`，如何实现多语言实时切换。
- [ ] **动态主题**：深入 CSS Variables 注入逻辑，实现运行时换肤。
- [ ] **全局方法封装**：研究 `ElMessage`、`ElMessageBox` 等命令式组件的单例模式与渲染机制。

---

## 🛠 第五阶段：工程化质量保障 (Engineering)

_目标：从“写完代码”到“高质量发布”_

- [ ] **单元测试**：使用 `Vitest` 和 `@vue/test-utils` 编写高覆盖率的测试。
- [ ] **视觉回归测试**：了解组件库如何保证样式不意外错乱。
- [ ] **自动化流水线**：了解 CI/CD、版本管理 (Changesets) 和 Changelog 自动生成。

---
