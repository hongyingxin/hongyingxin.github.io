# ai-i18n

这个是借鉴了[ai-api-docs](./ai-api-docs.md)的思路，用来生成国际化文件。

这里不需要脚本执行，所以不写`SKILL.md`文件。相当比较简单，所以直接在`.cursor/rules/i18n.mdc`文件中写规则。

在h5活动页面，这里的流程是让AI生产翻译，然后把翻译导出成特定格式`['中文', '英文', '阿语']`，然后复制到翻译文件中，由活动运营进行修改，然后再复制回来。

这里我们让AI以中文为key，按模块进行划分，复制回来的时候进行总结，罗列出新增/异常/修改的点。


## 实战效果

![实战效果](/public/assets/随笔/api-docs-1.png)

如上图所示，AI 在编码过程中，会根据规则来执行相应的命令，并输出结果。

然后会总结摘要，列出修改的点。

虽然这种方式跟直接给AI修改对比差不多，但行为预期是可控的。
