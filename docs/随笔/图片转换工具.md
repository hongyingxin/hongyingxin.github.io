# 图片转换

## 前言

webp格式比png格式的优势在于保持高画质的同时实现更极致的压缩，文件体积更小，网络加载极快，这也是前端优化的一个点。

## 在线拦截

我想实现一个“前端即时图片拦截与渐进式替换”的功能：页面代码里写的是` <img src="image.png">`，但希望浏览器在加载时，默默在后台把它转成 .webp 格式来减少流量，即在线转换功能。

核心技术：Service Worker

但这个过程遇到新问题：

1. 逻辑悖论：一开始我们的初衷是“为了节省用户流量，让加载变快”，但现在需要让用户先加载了图片，才能进行转换，相当于下载了两次图片。

2. 性能灾难：图片加载是网页渲染中最耗时的部份之一，加载 -> 拦截 -> 转换 -> 加载，这会导致页面出现长时间的白块或闪烁，LCP指标非常糟糕

3. 拦截的局限：Server Worker虽然可以拦截fetch请求，但内部没有DOM，需要引入PNG解码器和Webp解码器，脚本体积暴增

由于上面的问题，于是换了种做法

1. 利用`<picture>`标签

```html
<picture>
  <source srcset="image.webp" type="image/webp">
  <img src="image.png" alt="示例图片">
</picture>
```

这种做法带来另一个问题：包体积变大，因为同时保存了webp和png两种图片格式。

2. 内容协商

这种是通过配置第三方服务商（阿里云OSS、腾讯云COS、Cloudflare），返回我们想要的格式，但前提需要把图片放在CDN服务器上。

 - 页面代码里只需要写：`<img src="https://example.com/image.png">`

 - CDN 收到请求后，检查请求头，发现浏览器支持 WebP。

 - CDN 在服务器端自动将 image.png 转换为 image.webp（或者直接返回已经缓存好的 WebP 版本）。

 - 优点：前端代码一个字都不用改，流量真正变小，压力全在服务器/CDN，用户体验拉满。

## 脚本工具

写一个脚本，把png图片转换为webp图片。可以在本地手动转换或者集成到vite打包。

脚本转换涉及到两个核心底层：libvips和libwebp，目前市场上绝大多数库都基于这两者实现。

| 特性 | libvips | libwebp |
| --- | --- | --- |
| 核心定位 | 通用图像处理计算引擎 | WebP 格式官方编解码器 |
| 主要功能 | 缩放、裁剪、旋转、色彩空间转换、多图合成等 | 将常规图片压缩为 .webp，或将 .webp 解压 |
| 格式支持 | JPEG, PNG, WebP, AVIF, TIFF, SVG, GIF 等（全能） | 仅限 WebP |
| 内存占用 | 极低（基于流式处理和按需计算） | 取决于单张图片的大小 |
| 典型应用 | sharp (Node.js), image_processing (Ruby) | 各种浏览器、图片查看器、libvips 的底层依赖 |

libvips 是一个通用的图像处理库。它的最大特点是快且省内存。

libwebp 是由 Google 官方开发并维护的，专门用于编码（压缩）和解码（解压）WebP 格式图片的库。

因为追求画质的还原，避免在转换过程中透明png边缘问题以及画质降低，首先libwebp。
